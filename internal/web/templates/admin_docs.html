1对接官方支付

配置后，用户充值和对外 epay 下单都会调官方接口生成真实动态二维码（不再是静态收款码）。

支付宝（当面付）

  1. 登录 支付宝开放平台，创建/使用「当面付」应用。
  2. 在「配置 → 配置页」填入 应用 AppID。
  3. 下载/复制 应用私钥（PKCS#8 PEM），保存为文件，在配置页「密钥文件」区上传为 alipay_private.pem。
  4. 复制 支付宝公钥（PEM），上传为 alipay_public.pem（用于验签回调）。

微信支付（v3 Native）

  1. 登录 微信支付商户平台，开通 Native 支付。
  2. 在「配置页」填入 AppID、商户号 MchID、商户证书序列号。
  3. 设置 APIv3 密钥（32位），把该字符串保存为文件，上传为 wxpay_apiv3.key。
  4. 下载商户证书，取出 商户私钥（RSA2048 PEM），上传为 wxpay_private.pem。
回调基地址必须 https。在「配置页」填写后，系统用它拼接官方回调地址：{{if .Data.notifyBase}}{{.Data.notifyBase}}{{else}}<回调基地址>{{end}}/notify/alipay 和 /notify/wxpay。这两个地址也要填到支付宝/微信开放平台的回调配置里。

2对外提供 epay 协议（给下游商户）

在「商户管理」页面为每个下游商户建一条记录（pid + 密钥），对方凭此调用下面的接口。

接口地址

页面支付：{{if .Data.notifyBase}}{{.Data.notifyBase}}{{else}}<你的站点>{{end}}/submit.php
接口下单：{{if .Data.notifyBase}}{{.Data.notifyBase}}{{else}}<你的站点>{{end}}/mapi.php （返回 JSON，含二维码）
订单查询：{{if .Data.notifyBase}}{{.Data.notifyBase}}{{else}}<你的站点>{{end}}/api.php?act=order

下单参数

参数必填说明
pid是商户管理里分配的 PID
out_trade_no是商户订单号（唯一）
notify_url是支付成功后通知商户的地址
name是商品名
money是金额（元，如 10.00）
type否alipay（默认）/ wxpay
sign是MD5 签名（见下）

签名算法（MD5）

  1. 取所有非空参数（去掉 sign、sign_type），按 key 字典序排序。
  2. 拼成 k1=v1&k2=v2&...
  3. 末尾直接接商户密钥（不加 &），整体做 MD5，转小写十六进制。
money=10.00&name=测试¬ify_url=https://m.com/n&out_trade_no=O1&pid=1001&type=alipay<商户密钥>

回调通知（发给下游 notify_url）

支付成功后，系统向商户的 notify_url 发 GET 请求，带签名参数 trade_status=TRADE_SUCCESS 等。商户校验 sign 后回写 success 即完成。

去商户管理

3faka-site 自身充值

本站自己的 /recharge 充值，就是 epay 网关的「第一个内部商户」。只需：

  1. 在「商户管理」建一个商户，记下它的 PID。
  2. 在「配置页」把 自身充值商户 pid 填成那个 PID。
  3. 把官方支付（第一部分）配好。

用户充值时即生成官方真实二维码，支付后自动到账（幂等，不会重复加钱）。
